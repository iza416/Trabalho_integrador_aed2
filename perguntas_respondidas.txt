  Cadastro e atendimento, novas ações são adicionadas ao final com .append().
 Na função desfazer_ultima_acao(), o último elemento inserido é o primeiro a ser removido usando .pop().

Onde foi usada a hash table?
Foi usada na  tabela_hash_tipo. No código implementamos uma tabela hash tratamento por encadeamento, utilizamos índice de tamanho 10. A função calcular_hash(tipo_string) calcula o índice usando o resto da divisão (% 10). 

Qual algoritmo de ordenação foi implementado?
Inicialmente (no primeiro commit) utilizei o Bubble Sort, mas, após analisarmos o que foi explicado pelo professor em aula, concluímos que o Insertion Sort seria uma alternativa mais eficiente para o projeto.
O Insertion Sort funciona inserindo cada elemento na posição correta da parte da lista que já está ordenada. Para isso, uma variável é utilizada como chave e os elementos anteriores que possuem valor maior são deslocados uma posição para a direita até que seja encontrada a posição correta para a inserção. Essa lógica pode ser observada no trecho:

while j >= 0 and copia_lista[j]["id"] > variavel_chave["id"]
Esse algoritmo foi implementado na função ordenar_ocorrencias(), permitindo exibir as ocorrências ordenadas por ID, e também foi reutilizado na função gerar_relatorio_ordenado(), garantindo que o relatório gerado ao encerrar o sistema seja salvo com todas as ocorrências organizadas em ordem crescente de ID.

Qual estrutura foi mais adequada para busca rápida?
A Hash Table. Que nos permitiu organizar as ocorrências direto pelo tipo, reduzindo o tempo se tivéssemos que “ver” a lista geral inteira, elemento por elemento. 

Qual estrutura foi mais adequada para atendimento por prioridade?
Nós utilizamos uma fila comum por ordem de chegada.
